Declarable

Declarables in Angular 19 are a fundamental concept referring to components, directives, and pipes that must be declared in an NgModule before they can be used in templates. With the stabilization of standalone components in Angular 19, declarables can now also be self-contained without requiring NgModule declarations.


What are Declarables in Angular?

  • Definition: Declarables are Angular artifacts that can be used in templates. There are three types of declarables:
    • Components: Define views with their own templates, styles, and logic
    • Directives: Add behavior to existing DOM elements or components
    • Pipes: Transform data for display in templates
  • Declaration Requirement: In traditional Angular applications, all declarables must be declared in exactly one NgModule's declarations array before they can be used.
  • Scope: Declarables are only visible within the module they're declared in, unless explicitly exported and imported by another module.


Declarables in NgModules

  • Component Declaration: Components must be declared in an NgModule before they can be referenced in templates.

  • Directive Declaration: Directives must also be declared in an NgModule's declarations array.

  • Pipe Declaration: Pipes, like components and directives, must be declared in an NgModule.

Angular 19 and Standalone Components

  • Standalone Declarables: Angular 19 fully stabilizes standalone components, directives, and pipes, which do not require NgModule declarations.
    typescript
  • Import-based Architecture: Standalone components manage their dependencies through imports rather than module declarations, simplifying the architecture.
  • Coexistence: Angular 19 supports both NgModule-based declarables and standalone declarables in the same application, allowing for gradual migration.


Using Declarables in Templates

  • Template Usage: Once declared (or marked as standalone), components, directives, and pipes can be used in templates.

  1. Organize Declarables Logically
    • Feature Modules: Group related declarables into feature modules to improve organization and maintain separation of concerns.
    • Shared Modules: Place common declarables in shared modules and export them for use across the application.
  2. Prevent Duplicate Declarations:
    • Single Declaration Rule: Ensure each declarable is declared in exactly one NgModule to prevent runtime errors. Angular will throw an error if a component, directive, or pipe is declared in multiple modules.
    • Use Module Imports: Instead of re-declaring components across modules, import the module that declares and exports them.
  3. Consider Standalone Components:
    • For New Development: In Angular 19, prefer standalone components, directives, and pipes for new development, as they simplify the architecture and reduce boilerplate.
    • Gradual Migration: For existing applications, gradually migrate to standalone components starting with leaf components and moving toward the root.
  4. Export Public Declarables:
    • Explicit Exports: Only export declarables that are intended to be used outside their declaring module. Keep internal components private.
    • Export Re-exports: When creating aggregator modules, re-export declarables from imported modules to simplify the public API.
  5. Test Declarables in Isolation:
    • Component Test Harnesses: Use Angular's Component Test Harnesses to test components in isolation, without their parent module dependencies.
    • Shallow Component Tests: For unit tests, prefer shallow component tests that don't require child components to be declared or mocked.
Pros of NgModule Declarables:
  1. Explicit Component Boundaries
    • NgModule declarations clearly define which components, directives, and pipes are available within a module scope, making the boundaries explicit.
  2. Encapsulation
    • NgModules provide strong encapsulation, ensuring that declarables are only available where explicitly exported and imported.
  3. Lazy Loading
    • NgModules facilitate feature-based lazy loading, where entire feature sets of declarables can be loaded on demand.
  4. Established Pattern
    • The NgModule architecture is well-established with clear patterns and extensive documentation for organizing declarables.

Cons of NgModule Declarables:
  1. Boilerplate Code
    • NgModules add boilerplate through declarations, imports, and exports arrays that must be maintained as the application grows.
  2. Cognitive Overhead
    • Understanding which module declares a component and tracking the module dependency tree can increase cognitive load.
  3. Duplication Risk
    • The single-declaration rule can lead to errors when a declarable is accidentally included in multiple modules.
  4. Indirection
    • The layer of indirection between components and their dependencies can make code navigation and understanding more difficult.

Pros of Standalone Declarables (Angular 19):
  1. Simplified Architecture
    • Standalone components eliminate the need for NgModule declarations, reducing boilerplate and simplifying the application structure.
  2. Direct Dependencies
    • Dependencies are declared directly at the component level, making it clearer what each component needs.
  3. Tree-Shakable
    • Standalone components are more efficiently tree-shakable, potentially reducing bundle sizes.
  4. Easier Testing
    • Testing standalone components is simpler as they explicitly declare their dependencies, requiring less test setup.

Cons of Standalone Declarables:
  1. Dependency Management
    • Without NgModules, developers must be careful to import all necessary dependencies in each standalone component.
  2. Migration Complexity
    • Migrating existing applications from NgModule-based declarables to standalone components requires careful planning.
  3. Potential Duplication
    • Without the centralized declarations of NgModules, there's a risk of duplicating import statements across components.

Related Angular 19 Topics:

  1. Standalone Components:

    Angular 19 fully stabilizes standalone components, which eliminate the need for NgModule declarations. Standalone components can import their dependencies directly, including other standalone components, directives, and pipes.

  2. NgModules:

    While Angular 19 emphasizes standalone components, NgModules remain a fundamental part of Angular for organizing and encapsulating application features. Understanding the relationship between modules and declarables is still important.

  3. Components, Directives, and Pipes:

    These three types of declarables form the building blocks of Angular UIs. Components define views, directives modify DOM elements, and pipes transform data. All can now be either NgModule-declared or standalone.

  4. Lazy Loading:

    Angular 19 supports lazy loading for both NgModule-based features and standalone components. With standalone components, Angular introduces new, simpler patterns for lazy loading that don't require modules.

  5. Dependency Injection:

    Angular's dependency injection system works closely with declarables, especially for providing services to components. In standalone components, dependencies can be injected directly without module providers.
